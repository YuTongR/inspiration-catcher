# 存入灵感标题表单布局调整设计

## 背景

当前 `存入灵感` 页面左侧内容区有两处与文本录入标题相关的输入结构：

- 顶部使用 `录入方式` 标题和一个 `快速定位` 输入框
- `粘贴文字` tab 内又单独提供一个 `素材标题` 输入框

这会造成标题语义分散和重复填写。用户希望把顶部区域直接改造成正式的 `素材标题` 输入区，让它成为唯一标题入口，同时删掉下方重复标题区域，并适度放大正文输入框。

## 目标

- 将左侧顶部原 `录入方式` 区块改为 `素材标题`
- 顶部输入框改为唯一的正式标题输入框，作为文本素材入库标题来源
- 删除 `粘贴文字` tab 内原有的重复 `素材标题` 输入框
- 保留现有页面主结构、左侧深色导航栏、主要配色和双栏布局不变
- 将“填写灵感碎片”的正文输入框高度增加约 15%，并适度优化留白与表单清晰度

## 非目标

- 不改左侧导航栏
- 不改整体配色体系
- 不改 URL / 图片录入流程
- 不重构 `存入灵感` 页面为新的布局体系

## 方案选择

### 方案 A：顶部标题区承担唯一标题输入，正文区留在 `粘贴文字` tab 内

优点：

- 结构改动最小
- 标题职责清晰
- 与现有 tabs 关系自然
- 最符合“替换上方、删除下方重复”的需求

缺点：

- 标题输入与正文输入仍分属上下两个区块，但处于同一录入流程中

### 方案 B：把标题和正文都提升到 tabs 之上

优点：

- 文本录入最集中

缺点：

- 会破坏 URL / 图片 tab 的并列关系
- 改动范围偏大，不适合本次局部修复

本次采用方案 A。

## 详细设计

### 1. 顶部标题区替换

左侧顶部原 `录入方式` 区块保留原有标题层级、字重和整体视觉语言，但文案改为 `素材标题`。

原先位于该区块下方的输入框，不再承担 `快速定位` 语义，而是直接改为正式标题输入框：

- 输入键仍服务于文本素材录入
- 文案与占位提示要明确表达“这是入库标题”
- 保存文本素材时，继续使用该输入值作为 `save_material(..., content)` 的标题参数来源

### 2. 文本录入区去重

在 `粘贴文字` tab 中：

- 保留 `存入一段灵感碎片` 的标题和说明文案
- 删除当前重复的 `素材标题` 输入框
- 正文输入框继续作为主要内容录入区
- 保存按钮位置和行为保持不变

这样文本录入流程会变为：

1. 先在顶部填写 `素材标题`
2. 再在 `粘贴文字` tab 中填写正文内容
3. 点击 `存入素材库`

### 3. 正文输入框放大

当前正文输入框高度为 `320`。本次调整为约 `368`，增幅约 15%。

调整原则：

- 不改变当前列宽和页面整体骨架
- 不让正文框挤压右侧信息栏到失衡
- 通过更舒展的上下留白，让输入区更清晰、更易读

### 4. 样式边界

允许的样式调整仅限：

- 顶部标题文案替换
- 顶部输入框标签/占位文案优化
- 文本录入区文案与留白微调
- 正文输入框高度调整

不允许的样式调整：

- 修改左侧深色导航栏
- 修改整体背景、主色、卡片体系
- 调整双栏结构比例
- 改动其他页面的视觉语言

## 测试与验收

### 自动化回归

需要验证：

- 页面源码中顶部文案由 `录入方式` 变为 `素材标题`
- `粘贴文字` tab 内不再保留重复的 `素材标题` 输入块
- 文本保存逻辑仍通过顶部 `material_title` 状态写入标题

### 手工验收口径

- 左侧顶部首先看到的是 `素材标题` 区块
- 页面中不再出现重复的第二个标题输入框
- 正文输入框明显更大，但整体布局仍协调
- URL / 图片 tab 交互不受影响
- 左侧导航栏和整体页面配色无变化

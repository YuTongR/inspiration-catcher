# 灵感捕手搜索页设计说明

## 背景

当前项目已经完成 `存入灵感` 和 `素材知识库` 两个可演示页面，但还没有补上 `7/16` 目标中的 `搜索页`。现阶段 A、B 模块尚未完成联调，因此搜索页需要先以 C 模块可独立演示为目标实现，同时保留后续接入 B 模块真实搜索能力的接口边界。

本次设计只覆盖 C 模块中的 Streamlit 页面与前端搜索适配逻辑，不涉及数据库、向量库、API 调用或 B 模块实现。

## 目标

- 新增一个可切换的 `搜索页`
- 页面可基于现有前端素材数据独立完成搜索演示
- 搜索结果具备接近真实产品的结构，而不是纯占位空壳
- 为后续 B 模块 `search_materials(keyword)` 联调预留稳定的数据接口
- 保持当前项目已经建立的暖色后台视觉语言，不引入新的页面风格分叉

## 非目标

- 不接 DeepSeek、ChromaDB、SQLite 或任意后端服务
- 不实现真正的向量检索或语义检索
- 不新增 `撰写页`
- 不补 PPT 文件或演示脚本
- 不修改现有 `存入灵感` 和 `素材知识库` 的核心交互行为

## 方案选择

### 方案 A：只做搜索页空壳

优点：

- 实现最快
- 风险最低

缺点：

- 演示价值偏低
- 后续仍需再补搜索行为和结果展示
- 无法支撑 `7/16` 目标中的“搜一个关键词看到相关素材”

### 方案 B：直接在页面里写本地搜索逻辑

优点：

- 现在就能完成可演示搜索闭环
- 代码量适中

缺点：

- 搜索逻辑和页面耦合
- 后续接 B 模块时替换成本更高

### 方案 C：抽一个前端搜索适配函数，本地实现为 fallback

这是本次确认采用的方案。

优点：

- 当前可以独立演示
- 后续联调时只替换函数实现，不重做页面
- 页面结构和结果结构更接近正式版本

代价：

- 比直接写死在页面里多一层封装
- 需要提前约束结果字段

## 页面结构

侧边栏导航新增一个页面入口：

- `存入灵感`
- `素材知识库`
- `搜索页`

`搜索页` 延续现有统一视觉系统，包括：

- 暖色背景
- 深色侧边栏
- 圆角卡片
- 统一标题区和统计区

页面主体分为两部分：

1. 顶部搜索工作台
   - 搜索关键词输入框
   - `开始搜索` 按钮
   - 简短说明文案，说明当前页面用于从已收录素材中寻找相关内容

2. 下方结果区
   - 搜索前默认显示引导提示，不直接展示全部素材
   - 搜索后显示命中数量
   - 按卡片列表展示结果
   - 每张卡片显示标题、标签、摘要、日期、匹配说明和占位分数

## 数据流设计

新增一个前端适配函数：

`search_materials(query: str) -> list[dict]`

该函数只负责从 `st.session_state.materials` 中读取已有素材并返回统一结构的结果列表，不调用数据库或 B 模块接口。

函数输出的每条结果至少包含以下字段：

- `title`
- `tags`
- `summary`
- `date`
- `score`
- `match_reason`

后续 B 模块完成后，只需要将真实搜索结果适配为同样的字段结构，页面层无需改动。

## 本地搜索规则

当前版本的搜索规则保持简单、稳定、可解释：

- 标题命中权重最高
- 标签命中次之
- 摘要命中再次之

推荐采用基于命中位置的简单计分方式，例如：

- 标题命中加较高分值
- 标签命中加中等分值
- 摘要命中加较低分值

`match_reason` 需要向用户说明结果为什么出现，例如：

- `标题命中关键词`
- `标签与关键词相关`
- `摘要包含关键词`

搜索结果按 `score` 从高到低排序。

## 页面交互

- 用户输入关键词后，点击 `开始搜索` 才刷新结果
- 首次进入页面时，不展示全部素材，只展示引导说明
- 空关键词时，提示用户先输入关键词
- 无命中结果时，展示明确空状态
- 有命中结果时，展示结果数量和卡片列表

该页面的定位是“搜索入口”，不承担素材总览职责，因此不与 `素材知识库` 的默认浏览行为重复。

## 与 B 模块的联调边界

当前版本中，`search_materials` 是 C 模块的本地实现。

后续联调时，替换方式应为：

- 保留 `搜索页` 布局和交互不变
- 保留结果卡片渲染结构不变
- 将 `search_materials(query)` 的内部逻辑替换为 B 模块真实搜索调用
- 如果 B 返回真实相似度分数，可直接映射到 `score`
- 如果 B 返回命中说明或来源解释，可直接映射到 `match_reason`

这样可以确保联调阶段只改数据源，不改 UI 结构。

## 测试与验收

本次实现沿用当前项目的轻量 smoke test 风格，在 `test_app_smoke.py` 中补充以下断言：

- `搜索页` 文案存在
- `开始搜索` 文案存在
- `搜索结果` 或等价结果区文案存在
- `search_materials` 函数名存在

运行级验收口径限定为 C 模块可独立验证的部分：

- `streamlit` 应用可以启动
- 侧边栏可以切换到 `搜索页`
- 输入空关键词时出现提示
- 输入可命中的关键词时出现结果卡片
- 输入无命中的关键词时出现空状态

## 范围结论

这是一个面向 `7/16` 目标的前端独立补全项，重点是让 C 模块先具备完整的“搜索页演示能力”，并把与 B 模块的联调成本压缩到替换单个搜索适配函数的程度。

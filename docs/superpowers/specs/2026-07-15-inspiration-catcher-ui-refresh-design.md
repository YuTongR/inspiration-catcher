# 灵感捕手界面改版设计说明

## 背景

当前 Streamlit 应用已经具备基本可演示能力，但整体视觉仍偏早期原型。本次改版的目标是在不改动现有交互逻辑和数据流的前提下，把两个页面统一重做为简约、清新、轻量化的后台界面，用于 AI 素材管理产品 Demo 展示。

本次设计覆盖 `app.py` 中现有两个页面：

- `存入灵感`
- `素材知识库`

本次不涉及后端、数据库、页面路由或会话逻辑调整。

## 目标

- 保留当前所有已有交互能力
- 两个页面统一为同一套后台视觉语言
- 整体气质偏向知识管理、灵感笔记、AI 素材管家
- 满足以下明确视觉要求：
  - 奶油米白、浅暖底色
  - 深色左侧边导航栏
  - 圆角卡片
  - 低饱和暖色调
  - 留白充足
  - 无衬线字体，标题加粗，层级清晰

## 非目标

- 不接数据库
- 不修改 `session_state` 数据结构
- 不更改页面切换逻辑
- 不新增复杂功能流
- 不改动当前保存、搜索、筛选、预览的业务行为

## 方案选择

### 方案 A：保留现有交互，只重写布局和 CSS

这是本次确认采用的方案。

优点：

- 在 Streamlit 中实现风险最低
- 不影响现有演示流程
- 能较稳定地完成你要求的后台风格改版

代价：

- 部分原生控件细节仍会受 Streamlit 默认结构影响
- 页面视觉会更像成品，但底层仍是 Streamlit 组件体系

### 方案 B：更重度地伪装原生控件

优点：

- 理论上能更接近纯产品稿效果

代价：

- 脆弱度更高
- 更容易影响控件对齐、间距和稳定性
- 对当前 Demo 目标来说没有必要

## 视觉系统

### 整体布局

- 页面整体使用奶油米白暖底色
- 左侧为深色竖向侧边栏
- 主区域采用左主内容、右信息栏的双栏结构
- 各区块使用大圆角、轻阴影、细边框

### 色彩方向

- 页面背景：奶油米白、浅暖灰白
- 侧边栏：深炭灰、深蓝灰
- 主文字：深棕黑、深灰黑
- 辅助文字：暖灰色
- 当前导航高亮：克制的红色圆点
- 右侧信息面板：浅米色卡片

### 字体与层级

- 使用无衬线字体
- 页面大标题加粗
- 副标题、说明文字、标签、统计值形成清晰层级
- 不使用花哨字体效果

### 圆角与留白

- 头部信息区、输入区、素材卡片均采用圆角设计
- 大卡片圆角更大，信息面板和统计卡片圆角略小
- 通过较宽松的上下和左右留白建立安静、专业的后台气质

## 共用页面结构

两个页面都应共享统一的顶部结构：

1. 头部标题区
   - 标题：`灵感捕手 | Inspiration Catcher`
   - 副标题：`创作者的 AI 素材管家 —— 让收藏夹从"吃灰"变成"第二大脑"`
   - 一段项目说明文字

2. 横向三张统计卡片
   - `素材总数`
   - `标签种类`
   - `当前阶段`

3. 页面主体内容区
   - 根据页面不同承载输入区或素材库区

## 左侧边栏设计

左侧深色侧边栏包含两部分：

1. 顶部品牌区
   - 标题：`灵感捕手 Inspiration Catcher`
   - 一段简短项目简介

2. 下方导航区
   - `存入灵感`
   - `素材知识库`

当前页高亮方式：

- 使用红色圆点作为当前页标识
- 整体导航保持克制，不做多余装饰
- 当前页是最强视觉提示

## 页面设计：存入灵感

### 主内容列

自上而下布局如下：

1. 共用头部标题区
2. 共用统计卡片区
3. 中部单行简短输入框
4. 区块标题：`存入一段灵感碎片`
5. 一段辅助说明文字
6. 大尺寸白色圆角文本输入框
7. 保留现有主按钮用于保存

文本框需要内置灰色占位示例文字。

当前保存逻辑保持不变。

### 右侧信息列

右侧垂直排列三块浅米色圆角信息面板：

1. `输入建议`
2. `推荐素材`
3. `当前预览`

这三块面板以辅助阅读和输入为主，不应抢主输入区的视觉重心。

## 页面设计：素材知识库

素材知识库页面沿用相同的背景、侧边栏、头部标题区和统计卡片语言。

主体区重做以下部分的视觉：

- 搜索框
- 标签筛选区
- 素材卡片

素材卡片应清晰区分以下层级：

- 标题
- 标签
- 摘要
- 日期

页面气质保持轻量、温和、专业，避免做成内容过密的管理系统。

当前搜索和筛选逻辑不变。

## 交互约束

- 保留现有 `st.session_state` 结构
- 保留现有导航逻辑
- 保留现有保存、预览、筛选、搜索交互
- 仅调整文案到更符合本次界面的表达方式

## 实现说明

- 改动集中在 `app.py`
- 优先通过 CSS 覆盖和少量结构重排完成视觉改版
- 尽量保留现有示例数据和辅助函数
- 保证宽屏布局下两个页面都能稳定展示

## 验证要求

实现后需要确认：

- `streamlit run app.py` 可以正常启动
- 左侧导航仍可切换两个页面
- 保存素材功能仍可用
- 当前预览仍随输入变化
- 搜索和标签筛选仍可用
- 两个页面都统一成这次确认的后台风格

## 风险说明

- Streamlit 原生控件外层结构可能需要更精准的 CSS 选择器控制间距和边框
- 控件原生标签的默认布局可能需要额外处理，避免破坏极简外观
- 如果过度深改控件内部样式，页面稳定性会下降，因此视觉改版应以表层样式统一为主

## 范围结论

这是一次边界明确的单步 UI 改版，不需要再拆成多个独立子任务。
